description: Сгенерировать commentary.txt по result-файлу портфеля
---

Сгенерируй комментарий в формате **компактной внутренней analyst note** (институциональный, по делу, без «машинного compliance»).

Обязательный стандарт: `.cursor/rules/portfolio-commentary.mdc` — следуй ему по структуре, тону и запретам.

Перед началом:
1) Source of truth для метрик и политики:
   - `docs/metrics_specification.md` (или корневой `metrics_specification.md`, если есть)
   - `docs/portfolio_construction_policy.md`
   - `docs/data_policy_nan_young_etfs.md`
   - `docs/stress_testing_spec.md` (если есть)
2) Интерпретируй только то, что есть во входных файлах и в соседних артефактах прогона (stress, snapshot, RC, run_result и т.д.). Не выдумывай числа.
3) Не пиши buy/sell и не строй макро-истории без опоры на цифры из источников.
4) Не используй формулировки вроде «мало деталей / ограниченная прозрачность», если деталь можно взять из доступных файлов — извлеки её.
5) **Не добавляй** секцию «Final note», финальные дисклеймеры и шаблонные compliance-закрытия.

Вход:
- путь к одному result-файлу (`report.txt`, `summary.txt`, `ips_summary.txt`, `run_result.json` и т.п.)

Действия:
1) Прочитай входной файл; при необходимости дочитай связанные файлы в той же папке (stress_report, snapshot, csv метрик/RC).
2) Создай/обнови `commentary.txt` в той же папке.
3) Если в папке уже есть `commentary.txt` для другого source, добавь новый блок с первой строкой `Source: <имя_файла>`.

Структура (строго, порядок секций не менять):
- Executive Summary
- Metric-by-Metric Interpretation
- Risk Structure
- Strengths
- Weaknesses
- Scenario Behavior
- Final Conclusion

Содержание:
- **Executive Summary:** 3–5 содержательных предложений, не две общие фразы.
- **Метрики:** для ключевых — что показывают, что значат для профиля, по возможности связь с общей картиной и с контекстом сравнения (EW/RP/бенчмарк), если он есть во входе. Избегай однотипного «одно сухое предложение на метрику».
- **Risk Structure:** концентрация/диверсификация/уязвимость из RC_vol по блокам и активам, бета, корреляции, просадка, стресс — что явно следует из данных.
- **Strengths / Weaknesses:** конкретика по этому прогону, не общие шаблоны.
- **Scenario Behavior:** нормальные условия vs стресс — предметно по именам сценариев и статусам, если они в источнике.
- **Final Conclusion:** профиль портфеля и главный trade-off развёрнуто; без нового дисклеймера после секции.

Стиль (русский текст — грамотный, читаемый):
- лаконично, но не скелетно; без воды, без пустых оговорок и роботизированных формулировок.
