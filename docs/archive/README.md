# Archive Register

This directory stores archived documentation retained for traceability only.

Archived files may contain older product language, optimizer-first framing, old route assumptions,
legacy command behavior, historical screenshots, or superseded architecture plans. They do not
override `README.md`, `PRODUCT.md`, `SPEC.md`, `OUTPUTS.md`, `DATA.md`, `TESTING.md`,
`docs/contracts/*.md`, `docs/specs/*.md`, or current code.

Use archived files to understand why past decisions were made. Do not use them to describe current
product truth, runtime behavior, formulas, schemas, output contracts, QA requirements, or frontend
routes unless an active source-of-truth document confirms the same claim.
