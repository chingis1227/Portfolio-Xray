# Описание всех файлов

Ниже собраны все файлы, которые мы описывали в этом чате, без дубликатов и в едином порядке.

## Корень проекта

### `README.md`
Главная инструкция по запуску пайплайна: сначала оптимизация, затем отчёт. Также файл даёт карту ключевых конфигов и документов-источников правил.

### `PROJECT_RULES.md`
Свод проектных стандартов по метрикам, частоте данных, FX, бенчмаркам и стресс-тестам. Это практический регламент, который запрещает произвольные формулы и «ручные» отклонения от политики.

### `metrics_specification.md`
Формальная спецификация расчётов: определения метрик, выравнивание рядов, окна, правила `ddof=1`. На этот документ опирается вся математика метрик в коде.

### `requirements.txt`
Список Python-зависимостей с минимальными версиями для запуска проекта. Нужен для воспроизводимой установки окружения.

### `config.yml`
Основной рабочий конфиг текущего запуска: валюта, капитал, профиль, тикеры и выходные директории. Его читают оптимизация и отчётный пайплайн.

### `config.yml.example`
Расширенный шаблон конфига с пояснениями и примерами параметров. Удобен как стартовая заготовка для нового проекта или проверки структуры.

### `assets.yml`
Файл точечных переопределений метаданных активов (например, валюты). Если не заполнен, система пытается определить данные автоматически.

### ~~`blocks_universe.yml`~~ *(удалён)*
Ранее задавал соответствие тикеров блочной модели. В текущей версии универсум задаётся только списком **`tickers`** в `config.yml`; отдельного файла блоков нет.

### `.gitignore`
Определяет, какие файлы и папки не нужно коммитить в Git. Обычно исключает кэш, окружения, IDE-артефакты и сгенерированные результаты.

### `eua_volatility.py`
Отдельный скрипт для расчёта дневной и годовой волатильности EUA-фьючерса с Yahoo. Он не является ядром пайплайна портфельной оптимизации.

### `run_optimization.py`
Основной сценарий построения целевых весов: одностадийная оптимизация (max expected return + мягкие цели vol/return + per-asset RC cap), ProLiquidity, мандат MaxDD. Результат пишет в итоговые файлы и при стандартных настройках запускает отчёт.

### `run_report.py`
Сценарий формирования всех отчётов и экспортов: CSV/JSON/TXT/HTML, snapshot-ы, stress и аналитика. Работает на уже подготовленных весах.

### `run_equal_weight.py`
Строит baseline-портфель с равными весами по допустимому универсуму. Затем прогоняет его через тот же контур метрик и стресс-тестов.

### `run_risk_parity.py`
Строит baseline Risk Parity на уровне активов без policy-конструкции блоков. После этого формирует полный набор отчётов для сравнения.

### `run_compare_variants.py`
Сводит результаты Policy/Equal-Weight/Risk-Parity в единый JSON и текстовую таблицу. Нужен для быстрого сравнительного анализа вариантов.

### `run_rebalance.py`
Считает список сделок для перехода от текущих весов к целевым. Поддерживает пороги, чтобы не совершать лишние микро-сделки.

### `run_view_after_optimization.py`
Реализует протокол добавления PM-view после оптимизации через контролируемые гейты. Формирует прозрачный `view_execution_report.json` с исходом и причинами.

## Конфиги и UI

### `config/client_profiles.yml`
Шаблоны клиентских профилей риска и доходности (ultra_conservative ... aggressive). Из него автоматически подтягиваются midpoint-параметры в конфиг.

### `config_ui/app.py`
Flask-приложение для редактирования `config.yml` через веб-форму. Может валидировать конфиг и запускать оптимизацию из интерфейса.

### `results_dashboard/app.py`
Flask-дашборд (по умолчанию порт 5005): ключевые итоги последнего прогона в `output_dir_final` — `run_result.json`, `snapshot_10y.json`, ссылка на `report.html`. Стили: `DESIGN.md`, общий `config_ui/static/design.css` + `results_dashboard/static/dashboard.css`.

## Ядро `src/`

### `src/__init__.py`
Маркер Python-пакета `src`. Нужен для корректных импортов модулей.

### `src/config_schema.py`
Схема и строгая валидация полей конфига, включая значения по умолчанию и проверки совместимости. Содержит типизированный контейнер `PortfolioConfig`.

### `src/config.py`
Загрузка, объединение и нормализация конфигов/весов/профилей. Здесь же резолв cash/rf.

### `src/client_profiles.py`
Логика применения профилей клиента к конфигу (целевые vol/MaxDD/return, liquidity floor и т.д.).

### `src/optimization.py`
Одностадийная max-return оптимизация с мягкими штрафами по vol/return, RC-cap по активам и постобработкой. Главный «движок» целевых весов RiskPortfolio.

### `src/portfolio_variants.py`
Реализация baseline-портфелей (Equal-Weight и Risk-Parity) без policy-ограничений. Нужна для честного сравнения с Policy-портфелем.

### `src/portfolio_dynamic.py`
NaN-safe расчёт портфельной доходности по периодам с fallback в cash. Глобальное перераспределение среди risk-тикеров и RC-gated fallback в cash proxy.

### `src/rebalance.py`
Вычисляет дельты и формирует сделки buy/sell для ребаланса. Логику использует CLI-скрипт `run_rebalance.py`.

### `src/data_loader.py`
Общий загрузчик данных и кэш-контур: цены, FX-конверсия, month-end, returns, benchmark, rf и cash. Убирает дубли логики между `run_optimization.py` и `run_report.py`.

### `src/data_yf.py`
Загрузка дневных `Adj Close` из yfinance для активов, бенчмарков и FX. Возвращает унифицированные ряды цен для последующих шагов.

### `src/data_fred.py`
Загрузка risk-free/макро рядов из FRED и преобразование ставки в месячную эффективную форму. Используется как базовый источник rf для USD-сценариев.

### `src/data_ecb.py`
Загрузка €STR для EUR risk-free через API. Нужен для корректной работы в EUR-контуре.

### `src/fx.py`
Конвертирует цены активов в валюту инвестора по явным правилам FX-направления. Поддерживает инверсию пар и единый подход к кросс-валютам через USD.

### `src/resample.py`
Ресемплинг дневных рядов в эффективный month-end (последний торговый день месяца). Даёт согласованную частоту перед расчётом доходностей.

### `src/returns.py`
Расчёт простых и лог-доходностей, плюс построение equity curve. Базовый строительный модуль для метрик.

### `src/windows.py`
Определяет `analysis_end` и корректные срезы окон 36/60/120M. Гарантирует единообразие датовой логики во всём проекте.

### `src/metrics_asset.py`
Метрики по каждому активу: CAGR, vol, Sharpe/Sortino, beta, Treynor, skew/kurt, MaxDD, TTR. Реализует формулы из спецификации с корректным выравниванием рядов.

### `src/metrics_portfolio.py`
Портфельные метрики по окнам на той же методологии, что и метрики активов. Используется в отчётах и snapshot-артефактах.

### `src/risk_contrib.py`
Ковариации и RC_vol (вклад в дисперсию), включая агрегирование по окнам. Также содержит резолв per-asset RC cap по policy-формулам.

### `src/portfolio_analytics.py`
Расширенная аналитика: rolling Sharpe/Sortino/vol, drawdown structure, VaR/ES, EEE. Формирует диагностические временные ряды для отчётов.

### `src/stress_factors.py`
Подготавливает факторные ряды и оценивает факторные beta для stress-движка. Использует FRED и ETF-прокси на недельной частоте.

### `src/stress.py`
Сценарный стресс-тест с проверками Loss/Role/RC и кодами причин отказа. Возвращает итоговый статус и детализацию нарушений по сценариям.

### `src/robustness.py`
Диагностика устойчивости решения между горизонтами (обычно 10Y и 5Y). Выдаёт флаги нестабильности по весам и risk contribution.

### `src/snapshot.py`
Собирает итоговые snapshot-структуры и сводные отчёты (TXT/HTML/JSON). Это слой представления результатов для пользователя и интеграций.

### `src/io_export.py`
Экспортирует таблицы/JSON и сохраняет входные серии для воспроизводимости. Также формирует metadata и служебные итоговые файлы.

### `src/cache.py`
Двухуровневый кэш (daily/monthly) с ключами версий и метаданными. Ускоряет повторные прогоны и снижает нагрузку на источники данных.

### `src/utils.py`
Общие служебные функции: логирование, coverage-утилиты и проверочные helper-ы. Используется почти всеми модулями как инфраструктурный слой.

### `src/view_after_optimization.py`
Ядро логики post-optimization view: funding-порядок, gate-оценка, auto-shrink delta и итоговый отчёт. Гарантирует детерминированное применение view без ручной правки весов.

## Математические политики

### `policy_math/feasibility.py`
Централизованный модуль формул feasibility (RC caps, weight caps, достижимость RB). Нужен, чтобы правила ограничений были в одном месте и не расходились по коду.

## Тесты

### `tests/test_backtest_nan_safe.py`
Проверяет корректность NaN-safe логики и работу с young ETF без обрезки истории. Включает проверки перераспределения внутри блока и fallback в cash.

### `tests/test_optimization_fallback.py`
Проверяет, что оптимизатор возвращает валидные веса даже при жёстких ограничениях. Тест страхует от ситуации «оптимизация вернула пусто».

## Документация `docs/`

### `docs/portfolio_construction_policy.md`
Основной policy-документ архитектуры: роли блоков, иерархия ограничений и принципы принятия риска. Используется как источник истины для портфельной конструкции.

### `docs/data_policy_nan_young_etfs.md`
Правила обработки NaN и молодых ETF, включая динамический NaN-safe backtest. Фиксирует как считать честно, не переписывая историю.

### `docs/operational_runbook.md`
Практический регламент эксплуатации: когда перезапускать, что проверять, как трактовать статусы. Полезен для регулярной операционной работы.

### `docs/optimization_run_checks.md`
Список типовых проблем полного прогона и способ их диагностики. Помогает быстро найти источник ошибки в конфиге, данных или ограничениях.

### `docs/production_workflow.md`
Кратко фиксирует, какие гейты блокируют запись весов, а какие только выставляют флаги. Это документ про интерпретацию статусов в прод-потоке.

### `docs/refactor_summary.md`
Сводка проведённого рефакторинга и распределения hard/soft/report-only правил. Ускоряет понимание текущей архитектуры без чтения всего диффа.

## Спецификации `docs/docs/`

### `docs/docs/README.md`
Оглавление раздела частных спецификаций оптимизации и связанных протоколов. Используется как навигатор по узкоспециализированным документам.

### `docs/docs/stress_testing_spec.md`
Полная спецификация стресс-тестов: сценарии, pass/fail критерии, обязательные outputs и fail-коды. Определяет поведение стресс-модуля как стандарта.

### `docs/docs/feasibility_constraints_spec.md`
Формулы ограничений достижимости и концентрации (RC, weight caps, block achievability). На этот документ завязана feasibility-математика в коде.

### `docs/docs/optimization_growth_spec.md`
Правила оптимизации блока Growth: цель, запреты и рамки. Подчёркивает, что внутри Growth не нужно имитировать «защиту», а нужно корректно брать ростовой риск.

### `docs/docs/optimization_duration_spec.md`
Спецификация выбора состава Duration через кандидатный scoring. Закладывает проверку защитных свойств в стрессовых режимах.

### `docs/docs/optimization_inflation_spec.md`
Спецификация выбора состава Inflation по механизмам хеджирования инфляционных рисков. Определяет логику кандидатов и критерии отбора.

### `docs/docs/optimization_proliquidity_spec.md`
Детализирует ProLiquidity: life floor, vol-scaling cash, и alpha-shift при запрете cash. Это алгоритм технической сборки ликвидностной части портфеля.

### `docs/docs/view_after_optimization_spec.md`
Протокол «добавить view после оптимизации» как единственный разрешённый путь тильта. Описывает меню delta, funding-правила, гейты и формат отчётности.

Snapshot для 10-летнего окна, обычно ключевой для стратегических выводов. Используется как основа долгосрочной оценки портфеля.

### `ФИНАЛЬНЫЕ РЕЗУЛЬТАТЫ/risk parity portfolio/drawdown_structure_10y.json`
Структура просадок для 10-летнего окна. Важна для долгосрочной оценки скорости и тяжести восстановлений.

### `ФИНАЛЬНЫЕ РЕЗУЛЬТАТЫ/risk parity portfolio/ips_summary.txt`

### `ФИНАЛЬНЫЕ РЕЗУЛЬТАТЫ/risk parity portfolio/portfolio_weights.yml`
Целевые веса в YAML для практического использования (ребаланс/исполнение). Обычно это основной файл «веса к действию».

### `ФИНАЛЬНЫЕ РЕЗУЛЬТАТЫ/risk parity portfolio/robustness_report.json`
Результаты dual-horizon проверки устойчивости (обычно 10Y vs 5Y). Подсвечивает, насколько решение стабильно при смене режима данных.

### `ФИНАЛЬНЫЕ РЕЗУЛЬТАТЫ/risk parity portfolio/run_metadata.json`
Технический паспорт запуска: время, resolved config, assumptions и служебные поля. Нужен для воспроизводимости и аудита.

### `ФИНАЛЬНЫЕ РЕЗУЛЬТАТЫ/risk parity portfolio/run_result.json`
Главный статусный файл прогона: итоговый `status`, нарушения, рекомендации и веса/причины их отсутствия. Это первый файл, который обычно смотрят после запуска.

### `ФИНАЛЬНЫЕ РЕЗУЛЬТАТЫ/equal-weight portfolio/weights.txt`
Итоговые равные веса baseline-портфеля (equal-weight). Используется как контрольная точка для сравнения с policy/risk parity.
